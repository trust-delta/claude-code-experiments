# Experiments Index

Claude Codeの機能に関する実験のインデックスです。

## 📂 ディレクトリ構造

各実験は以下の構造に従います：

```
XX-experiment-name/
├── README.md           # 実験の概要、目的、仮説
├── setup.md           # 環境構築手順
├── analysis.md        # 結果分析
└── [実験固有のディレクトリ/ファイル]
```

## 🔤 命名規則

### ディレクトリ名

`XX-short-descriptive-name`

- `XX`: 2桁の連番（01, 02, 03...）
- `short-descriptive-name`: ケバブケースの短い説明的な名前

**例**:
- `01-subagent-dialog-tasks`
- `02-skill-composition`
- `03-slash-command-patterns`

### ファイル名

- `README.md`: 実験の詳細な説明
- `setup.md`: セットアップ手順
- `analysis.md`: 結果分析
- その他: 実験内容に応じて自由に追加

## 🆕 新しい実験の追加

1. **テンプレートをコピー**:
   ```bash
   cp -r template XX-your-experiment-name
   cd XX-your-experiment-name
   ```

2. **README.mdを編集**:
   - 実験の目的を明確に記載
   - 仮説を立てる
   - 検証項目を列挙

3. **setup.mdを記載**:
   - 必要な環境・ツールを記載
   - セットアップ手順を詳細に記述

4. **実験を実行**:
   - 計画に従って実験を実行
   - ログやスクリーンショットを記録

5. **analysis.mdに結果を記載**:
   - 観察された挙動
   - 仮説との比較
   - 得られた知見
   - 今後の課題

6. **このファイルを更新**:
   - 実験一覧に新しいエントリを追加

## 📊 実験カテゴリ

実験は以下のカテゴリに分類されます：

- **サブエージェント**: サブエージェントの挙動や最適な使用場面
- **スキル**: スキルの構成、再利用パターン
- **スラッシュコマンド**: カスタムコマンドの実装パターン
- **MCP**: 外部ツール統合の実践
- **統合**: 複数機能を組み合わせた複雑なワークフロー

## 🎯 実験の品質基準

良い実験には以下の要素が含まれます：

1. **明確な目的**: 何を検証したいのか明確
2. **具体的な仮説**: 予想される結果を事前に記載
3. **再現可能性**: 他の人が同じ実験を再現できる
4. **定量的/定性的データ**: 客観的な評価基準
5. **学び**: 得られた知見と今後のアクション

## 📝 テンプレートの使用

新しい実験を始める際は、[template](template/)ディレクトリをコピーして使用してください。テンプレートには以下が含まれます：

- 実験設計のガイドライン
- 標準的なセクション構成
- 記録すべき情報のチェックリスト
