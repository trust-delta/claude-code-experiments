# 実験ガイド

## 実験の目的

Claude Codeの各機能（サブエージェント、スキル、カスタムコマンド）の挙動を体系的に検証し、効果的な使用方法を明らかにする。

## 実験の設計原則

### 1. 比較可能性

同一タスクを複数のアプローチで実施し、結果を比較可能にする。

### 2. 再現性

実験手順を明確に文書化し、他者が再現できるようにする。

### 3. 客観性

主観的な評価を避け、観察可能な事実に基づいて分析する。

## 実験テンプレート

各実験は以下の構造を持つ：

```
expXX-experiment-name/
  README.md           # 実験の概要
  .claude/            # 実験用の設定
  pattern-a/          # パターンA
  pattern-b/          # パターンB
  pattern-c/          # パターンC
  setup.md            # セットアップ手順
  analysis.md         # 分析結果
```

## 実験の実施手順

1. **仮説の設定**: 検証したい仮説を明確にする
2. **環境の準備**: .claude/配下に必要な設定を配置
3. **パターンの実施**: 各パターンで同一タスクを実行
4. **観察と記録**: 挙動を詳細に記録
5. **分析**: 結果を比較し、仮説を検証
6. **結論**: 推奨事項をまとめる

## 評価観点

### サブエージェント

- コンテキスト分断の影響
- タスクの独立性
- 出力の統合方法

### スキル

- 呼び出しの容易さ
- コンテキストの共有
- 柔軟性

### カスタムコマンド

- プロンプトの展開方法
- ワークフローの自動化
- 再利用性
