# 実験02: 対話的タスクにおけるサブエージェントの挙動検証

## 実験環境

- **実験日**: 2025-11-23
- **Claude Code バージョン**: (実験時に記入)
- **Claude モデル**: Claude Sonnet 4.5

## 目的

サブエージェントが対話的タスク（要件定義）に適しているか検証する

## 仮説

サブエージェントの独立コンテキストは対話的タスクに不向き
- 対話の非効率性
- セッション分断によるコンテキスト喪失
- 修正時の困難さ

## タスク

TODOアプリの「優先度自動調整機能」の要件定義

## 検証パターン

### パターンA: サブエージェント
- requirements-definerサブエージェントを使用
- 対話的に要件定義
- 完成後に修正を依頼
- コンテキスト分断の影響を観察

### パターンB: メイン + スキル
- メインでspec-helperスキルを使用
- 対話的に要件定義
- 完成後に修正を依頼
- 修正の容易さを観察

### パターンC: メイン単体
- スキルなしで要件定義
- 対話的に要件定義
- 完成後に修正を依頼
- ベースラインとして比較

## 検証項目

1. 対話フローの効率性
2. コンテキスト分断の影響
3. 修正時の柔軟性
