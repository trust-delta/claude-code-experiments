# 実験02: 対話的タスクにおけるサブエージェントの挙動検証

## 実験環境

- **実験日**: 2025-11-29
- **Claude モデル**: claude-opus-4-5-20251101

## 目的

サブエージェントが対話的タスク（要件定義）に適しているか検証する

## 仮説

サブエージェントの独立コンテキストは対話的タスクに不向き
- 対話の非効率性
- セッション分断によるコンテキスト喪失
- 修正時の困難さ

## タスク

TODOアプリの「優先度自動調整機能」の要件定義

## 検証パターン

### パターンA: サブエージェント
- requirements-definerサブエージェントを使用
- 対話的に要件定義
- 完成後に修正を依頼
- コンテキスト分断の影響を観察

### パターンB: カスタムスラッシュコマンド
- メインでrequirements-helperコマンドを使用
- 対話的に要件定義
- 完成後に修正を依頼
- 修正の容易さを観察


## 検証項目

1. 対話フローの効率性
2. コンテキスト分断の影響
3. 修正時の柔軟性

## 結果

詳細な分析結果は [analysis.md](./analysis.md) を参照してください。

### 主な発見

1. **セッション分断**: サブエージェントは各Task呼び出しで自動終了
2. **設計思想の違い**: サブエージェントは網羅的、カスタムコマンドは柔軟
3. **対話フローの違い**: サブエージェントは中継コストが高い、コマンドは自然な対話
4. **メインの自己評価**: サブエージェント経由でも「対話的タスクには不向き」と結論
