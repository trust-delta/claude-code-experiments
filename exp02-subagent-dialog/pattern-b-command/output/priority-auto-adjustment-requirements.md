# 優先度自動調整機能 要件定義書

作成日: 2025年11月29日

## 1. 概要

### 1.1 機能名
優先度自動調整機能

### 1.2 目的
ユーザーが手動で優先度を更新する手間を省き、タスク管理の効率化を図る。

## 2. 機能要件

### 2.1 自動調整のトリガー
- **期限ベース**: タスクの締切日までの残り日数に応じて優先度を自動調整する

### 2.2 優先度レベル（4段階）

| レベル | 名称 | 説明 |
|--------|------|------|
| 1 | 低 | 余裕がある状態 |
| 2 | 中 | 計画的に進める必要がある状態 |
| 3 | 高 | 早急に対応が必要な状態 |
| 4 | 緊急 | 即時対応が必要な状態 |

### 2.3 自動調整ルール

| 残り日数 | 優先度 |
|---------|--------|
| 7日以上 | 低 |
| 3〜6日 | 中 |
| 1〜2日 | 高 |
| 当日・期限超過 | 緊急 |

### 2.4 特殊ケースの扱い

| ケース | 動作 |
|--------|------|
| ユーザーが手動で優先度を設定した場合 | 手動設定を優先し、自動調整しない |
| 手動設定をリセットした場合 | 自動調整を再開する |
| 期限が設定されていないタスク | 優先度「低」のまま固定 |

## 3. 非機能要件

### 3.1 開発方針
- **シンプルさを重視**: 最小限の機能で早期リリースを目指す
- ルールのカスタマイズ機能は初期リリースに含めない

## 4. 将来の拡張候補（対象外）

以下は初期リリースには含めず、将来的な拡張として検討する：
- 調整ルールのユーザーカスタマイズ機能
- 時間経過ベースの自動調整
- 依存関係ベースの自動調整
- カテゴリベースの自動調整

## 5. 制約事項

- 優先度の自動調整は期限ベースのみとする
- 優先度レベルは4段階固定とする
